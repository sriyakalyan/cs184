Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

de Casteljau’s algorithm evaluates a point on a Bezier curve by repeatedly performing linear interpolation between neighboring control points. Given a parameter t between 0 and 1, the algorithm gives intermediate points by interpolating between each pair of adjacent control points. In simple terms, the parameter t defines how much weight is given to the first point. These intermediate points form a new, smaller set of points that represent the next level of the construction.

The interpolation process is repeated recursively. At each level, the number of points decreases by one because each new point is computed from a pair of points in the previous level. This continues until only a single point remains. That final point represents the position on the Bezier curve corresponding to the chosen parameter value t. A t value of 0 would return the first point, and a t value of 1 would return the second point.

In my implementation, I followed this iterative process by computing one level of interpolated points at a time. Starting from the original control points, I generated a new list of points by linearly interpolating between each neighboring pair using the parameter t. I then repeated this procedure on the newly generated list until only one point remained. This final point was returned as the evaluated position on the curve.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To compute area-weighted vertex normals, I iterated over all faces adjacent to a given vertex using the halfedge structure. Starting from one outgoing halfedge of the vertex, I traversed around the vertex by repeatedly following the twin and next pointers until I returned to the original halfedge. This allowed me to visit each neighboring triangle exactly once.

For each adjacent triangle, I retrieved the positions of its three vertices and computed two edge vectors originating from the current vertex. I then calculated the cross product of these two edge vectors to obtain a vector perpendicular to the triangle’s surface. The magnitude of this cross product is proportional to the area of the triangle, which weighs larger faces more heavily.

I accumulated these cross product vectors for all neighboring triangles, ignoring boundary faces. The resulting sum represents the area-weighted average of the surrounding face normals. Finally, I normalized this summed vector to produce a unit normal vector for the vertex.

Part 4: Edge flip

To implement the edge flip operation, I first ensured that the edge was not on the boundary, since boundary edges cannot be flipped. I then gathered all elements around the edge: the two triangles that share the edge, their six halfedges, the four vertices forming the quadrilateral, the five edges involved, and the two faces.

The flip operation replaces the diagonal of the quadrilateral formed by the two adjacent triangles. After finding the four vertices, I updated the halfedge relationships so that the shared edge connected the two opposite vertices instead of the original pair. I reassigned the next, twin, vertex, edge, and face pointers for all six halfedges to reflect the new arrangement. I then updated the halfedge references stored in the affected vertices, edges, and faces so that every element pointed to a valid halfedge in the new configuration.

Part 6: Loop subdivision for mesh upsampling

I implemented Loop subdivision by first computing new positions for all original vertices using the Loop subdivision rule. For each vertex, I looked at its neighboring vertices and calculated a weighted average of their positions. The weights depend on the number of neighboring vertices so that the new position smoothly reflects the surrounding geometry. I stored these new positions separately so the original vertex positions would not change during the intermediate steps.

Next, I computed the positions of the new vertices that would be created when splitting edges. For each edge, I calculated a new position using the Loop subdivision formula, which combines the positions of the edge’s two endpoints and the two opposite vertices from the neighboring triangles. I stored these positions in the edge so they could be used later when the edge was split.

After computing these positions, I split every edge in the original mesh. Splitting an edge creates a new vertex at the previously computed position. I also marked which vertices and edges were newly created so they could be handled differently in later steps. Finally, I flipped any new edge that connected an old vertex and a new vertex so that the mesh followed the correct triangulation for Loop subdivision.

Finally, I updated all vertex positions by copying the previously computed new positions into the actual vertex position field. This produced the refined mesh for the current subdivision step.

I added flags to the edge splitting method from the last part to keep track of which edges were newly created.

When repeatedly applying Loop subdivision to meshes, sharp corners and edges tend to become smoother and more rounded. This occurs because vertex positions are repeatedly averaged with neighboring vertices, which gradually removes sharp features. As a result, hard edges soften and corners become more curved as subdivision continues.

When applying Loop subdivision to a cube, I observed that the shape gradually becomes more rounded and slightly asymmetric after several iterations. This asymmetry happens because the initial triangulation of each square face of the cube introduces diagonals that break the symmetry of the mesh.

To improve the symmetry of the result, I preprocessed the cube by splitting the edges on each face so that a vertex existed at the center of the face. This created a more balanced triangulation where the faces were subdivided more evenly. With this preprocessing step, the subdivision process distributed new vertices more symmetrically across the cube’s surface, which helped reduce the asymmetry that appeared during repeated subdivisions.
